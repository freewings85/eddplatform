# AI 开发迭代管理系统：按模块开源选型清单 + "串一起"集成方案

> 版本 v1.0 ｜ 日期 2026-07-02 ｜ 负责人：AI 团队开发负责人
> 证据来源：第 4、5 轮 deep research（对抗式交叉验证）。关联：`开发迭代管理系统_选型与参考架构.md`（本文是它的"每模块清单 + 集成细化"版）。
> **标注约定**：✅=本轮/前轮经验证；🕯️=候选，本轮未验证（≠不合适，需 POC 再定）。

## 摘要：怎么"串一起"（先给答案）

各模块能拼成一体，靠的是 **4 个厂商中立的标准接口**（不用自己发明协议）：

| 胶水标准 | 打通哪两层 | 状态 |
|---|---|---|
| **OpenTelemetry `gen_ai.*` 语义约定** | 埋点 ↔ 评估/可观测（且**覆盖 MCP**，trace 可在任意后端间移植） | ✅ 官方标准，但 GenAI/MCP 约定仍 experimental |
| **OpenFeature + OFREP** | 应用代码 ↔ 灰度/flag（换 flag 后端**不改代码**） | ✅ CNCF incubating |
| **Apache DevLake** | ALM/CI/代码工具 ↔ 度量看板（汇聚成 DORA） | ✅ Apache 项目 |
| **REST API / webhook（+ MCP）** | ALM 事件 ↔ 自动化胶水 | ✅ 通用 |

**推荐可自托管组合（每层都能通过上面标准互通）**：
`pydantic-ai + Logfire(OTel)` → **Langfuse 或 Opik**（评估+迭代+trace 核心）＋ **OpenFeature/OFREP + Flipt/GrowthBook**（灰度）＋ **Plane/OpenProject/Vikunja**（ALM，webhook）＋ **BookStack**（文档）＋ **Apache DevLake→Grafana**（DORA）＋（可选）**Backstage**（golden path/软件目录）。

## 按模块开源清单

### 模块 1 · 需求/项目管理（ALM）
| 系统 | 能力 & 集成 | License/自托管 | 结论 |
|---|---|---|---|
| **Plane** ✅ | 需求·任务·周期·里程碑；**MCP server(100+工具)** + REST + OAuth(含 Agent Run scope) + webhook(HMAC) | MIT，自托管 | **首选**：API/MCP 最全，易接 eval |
| **OpenProject** ✅ | 需求·工作包·发布；REST APIv3(个人token,CORS) + 出站 webhook(项目/工作包/评论/工时/附件) | GPL，自托管 | 成熟稳，webhook 全（CORS 默认关）|
| **Vikunja** ✅ | 任务管理；REST 管理的出站 webhook(项目级+用户级)，HMAC-SHA256 签名，payload={event_name,time,data} | 开源，自托管 | 轻量，webhook 干净 |
| GitLab CE | 需求↔测试可追溯真有，但**仅 Ultimate 付费**、且 legacy 迁移中 | 开源核心 | 不建议纳入开源核心 |
| Taiga / Redmine / Leantime / Huly / Focalboard 🕯️ | 未验证 | — | 候选，需 POC |

### 模块 2 · 设计/知识/文档
| 系统 | 能力 & 集成 | License | 结论 |
|---|---|---|---|
| **BookStack** ✅ | Shelves/Books/Chapters/Pages，WYSIWYG+MD，全文搜索，内置 diagrams.net，角色权限 | **MIT，完全开源无付费门槛** | **首选**：承载设计文档/规范/复用登记 |
| Outline / Wiki.js / Docusaurus 🕯️ | 未验证 | — | 候选 |

### 模块 3 · 代码/CI + prompt 版本管理
- **代码托管**：Gitea/Forgejo、GitLab CE 🕯️（通识，未专门验证）
- **CI**：Woodpecker 🕯️
- **prompt 版本管理**：放进评估枢纽——**Langfuse prompts ✅ / Agenta ✅** 都自带；代码进 Git。
> 建议：prompt/上下文当"配置制品"在评估枢纽里版本化，代码在 Git，二者用 CI 串。

### 模块 4 · 评估/数据集/实验/LLMOps（★ 核心枢纽）
**平台（自托管，可当核心枢纽）：**
| 系统 | 能力 | License/注意 | 
|---|---|---|
| **Langfuse** ✅ | trace + eval + **prompt 版本** + 数据集 + LLM-judge + 标注 + Playground；原生吃 pydantic-ai OTel | **开源核心**：核心功能 MIT 无限制；企业设施(SCIM/审计/RBAC/保留)在 /ee 需 license。自托管 OTLP 需 v3.22+ |
| **Opik**(comet-ml) ✅ | 同类全套：Playground/数据集+实验/trace/LLM-judge/**人工标注**/CI(PyTest)；REST + Py/TS SDK + 原生 OTel | Docker/K8s Helm 自托管；⚠️**确切 license 本轮未定**(Apache-2.0 说法被证伪)，采用前先核实 |
| **Agenta** ✅ | prompt 管理/Playground、可观测、自动+人工评估、自定义工作流 | **开源核心**：功能特性 MIT；企业设施(RBAC/SSO/审计)商业 |
| **MLflow 3.x** ✅ | eval + tracing；OTLP `/v1/traces` 接入 + `gen_ai.*` 导出 | **Apache-2.0**；导出需 3.11+/接入需 3.6.0+，需 SQL 后端 |

**单点评测库（补充，进 CI 跑）🕯️**：Arize Phoenix（平台，OSS）、promptfoo、DeepEval、Ragas；**标注**：Argilla / Label Studio；Inspect(UK AISI)——均本轮未验证，作候选。
> **推荐**：核心枢纽选 **Langfuse**（或 Opik），把 promptfoo/DeepEval 之类当 CI 里的补充评测库（待 POC）。

### 模块 5 · 可观测/Trace
- **标准**：**OpenTelemetry `gen_ai.*`** ✅——span/metric/event，且**覆盖 MCP**（agent 侧与 MCP server 侧 trace 靠上下文传播打通），任意 OTLP 后端可收。
- **LLM 原生后端**：Langfuse / Opik ✅（复用评估枢纽即可）/ Arize Phoenix 🕯️
- **通用后端**：SigNoz / Grafana(Tempo) / Jaeger / Uptrace 🕯️；埋点侧 Traceloop OpenLLMetry 🕯️
> **推荐**：直接复用评估枢纽（Langfuse/Opik）当 trace 后端；用 **OTel Collector** 做路由与后端可换。

### 模块 6 · 发布/灰度/Feature Flag
- **标准**：**OpenFeature（CNCF incubating）+ OFREP（REST 协议）** ✅——Provider 是翻译层，换 flag 后端**一行注册、不改业务代码**；Unleash/GrowthBook/Flagsmith/Flipt 都有 provider。
- **后端**：**GrowthBook** ✅（MIT 开源核心，含统计实验引擎 CUPED/Bayesian/Bandit）、**Unleash** ✅（AGPL-3.0，金丝雀/渐进放量/AB/kill switch）、**Flipt** ✅（OFREP 采纳者）、Flagsmith ✅（OpenFeature 兼容）
> **推荐**：应用侧用 OpenFeature SDK，后端选 Flipt 或 GrowthBook；prompt/agent 版本按 flag 灰度。

### 模块 7 · 度量看板（DORA / DX）
- **Apache DevLake** ✅——连接 GitHub/GitLab/Jenkins/Jira/SonarQube 等 + webhook，汇聚成统一 SDLC 模型，算 **DORA 四指标**并出 **Grafana 看板**（SQL 可定制）。
- Grafana / Metabase / Superset 🕯️（看板层）
> **推荐**：**DevLake→Grafana** 出 DORA；DX Core 4 的自评/复用率等再叠自定义面板。

### 模块 8 · IDP / golden path / 脚手架
- Backstage（软件目录 + scaffolder 模板）、Copier / Cookiecutter（脚手架模板）、Kusion 🕯️——**本轮均未验证**。
> **推荐**：值得单独评估 Backstage（承载"批量造 agent"的软件目录 + golden path）+ Copier（脚手架），把 SOP §7.5 的标准脚手架落进去。

### 模块 0 · 埋点（Instrumentation，前轮）
- **pydantic-ai + Logfire** ✅——出 OTel `gen_ai.*`；`send_to_logfire=False` + `OTEL_EXPORTER_OTLP_ENDPOINT` 可改任意后端，不锁定。

## "串一起"集成方案（数据怎么流）

```
   ┌─────────────── ALM: Plane/OpenProject/Vikunja ───────────────┐
   │        需求·任务·发布（REST + webhook + MCP）                 │
   └───────┬──────────────────────────────────────────────────────┘
           │ webhook/REST（需求↔eval 绑定；发布事件）
   ┌───────▼──────── 薄胶水层（自研，唯一自研）────────────────────┐
   │  需求 → eval 数据集 → 回归运行 → eval 门控 → 灰度放量  可追溯   │
   └──┬───────────────────┬────────────────────┬───────────────────┘
      │ OTel gen_ai.*      │ eval/prompt API    │ OpenFeature/OFREP
   ┌──▼──────────┐   ┌─────▼───────────┐   ┌────▼────────────┐
   │ 埋点         │   │ 评估+迭代+trace  │   │ 灰度/flag        │
   │ pydantic-ai │──▶│ Langfuse / Opik  │   │ OpenFeature +    │
   │ + Logfire   │   │ (prompt版本/数据集│   │ Flipt/GrowthBook │
   │ (OTel)      │   │ /eval回归/judge/ │   │                  │
   │             │   │ 标注/Playground) │   └──────────────────┘
   └─────────────┘   └──────────────────┘
      │ OTel(gen_ai + MCP) 也喂给通用后端(SigNoz/Grafana/Jaeger 可选)
      ▼
   ┌───────────────── 度量看板: Apache DevLake → Grafana ──────────┐
   │  从 ALM/CI/代码工具 API 汇聚 → DORA 四指标 + DX 面板           │
   └───────────────────────────────────────────────────────────────┘
   文档/知识: BookStack（设计文档·规范·复用登记）｜脚手架: Backstage/Copier(待评)
```

**四条标准接口各自的职责**：
1. **OTel `gen_ai.*`**：埋点→评估/可观测后端；覆盖 MCP，所以 agent↔工具的调用也在同一套 trace 里；后端可换不锁定。
2. **OpenFeature/OFREP**：应用→灰度后端；换 Flipt/GrowthBook/Unleash 不改代码。
3. **Apache DevLake**：ALM/CI/代码→DORA 看板；一个平台把碎片数据汇成度量。
4. **REST/webhook（+MCP）**：ALM 事件→胶水层（Plane/OpenProject/Vikunja 都验证有 webhook）。

---

## 推荐组合（一句话）
**评估枢纽 Langfuse（或 Opik）+ 埋点 pydantic-ai/Logfire(OTel) + 灰度 OpenFeature→Flipt/GrowthBook + ALM Plane/OpenProject/Vikunja + 文档 BookStack + 度量 DevLake→Grafana**，用 **OTel + OpenFeature + DevLake + REST/webhook** 四标准串一起，自研只写"薄胶水"。

## 落地 MVP（最小可跑）
1. 自托管 **Langfuse**（≥v3.22）+ pydantic-ai `instrument=True` → 立即有 trace + prompt 版本 + eval 数据集 + LLM-judge。
2. 接 **OpenFeature SDK + Flipt/GrowthBook** → prompt/agent 版本可灰度、可 A/B。
3. 接 **Plane/OpenProject/Vikunja** + 写薄胶水（webhook：需求↔eval；eval 门控↔flag 放量）。
4. 起 **Apache DevLake→Grafana** → DORA 看板。
5. **BookStack** 放设计文档/规范；固化进 golden path 脚手架。
